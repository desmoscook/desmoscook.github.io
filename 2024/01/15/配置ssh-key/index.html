github配置ssh-key

desmoscook
2024-01-15

SSH 密钥是一种用于认证的公钥加密方法，常用于安全地连接到远程服务器。使用 github 的小伙伴一定非常熟悉这个名词，因此 github 最常用的认证方式就是 SSH

这里记录一下配置 SSH 密钥的详细步骤，包括生成密钥对、将公钥添加到远程服务器、测试连接等。

密钥对生成和添加

1. 生成 SSH 密钥对

下面命令生成密钥，默认保存在 ~/.ssh/id_rsa。对于每个密钥，会有一个私钥和一个公钥，分别为 id_rsa 和 id_rsa.pub

  • -t：执行密钥类型
  • -C：为密钥加一个标签（通常是电子邮件地址）
  • -f：指定文件的路径和名称。如果要配置多个ssh-key，如github一个gitlab一个，可以使用 -f 指定

2. 添加 SSH 公钥

进入 github，在 settings -> SSH and GPG keys 中，点击 new SSH key，可以看到我们需要填写 key 字段。而这里的 key 字段，实际上就是我们生成的公钥。

常见问题

1. Permission denied (publickey).

问题排查思路

  1. 首先检查github配置的rsa key是否正确
  2. git config user.name检查当前git的用户名是否是github的用户名。对于配置了多个公钥的机器，可能曾使用过 git config --global 来全局配置 name 和 email，导致覆盖了 git 仓库的 name 和 email 而连接失败
  3. 检查~/.ssh/config文件，默认连接使用的密钥是id_rsa，如果github使用的公钥不是id_rsa，则需要在SSH config文件里进行配置
