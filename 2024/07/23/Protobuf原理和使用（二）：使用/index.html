1. 定义 Protobuf 数据结构（IDL）

Protobuf 的 IDL 文件以 .proto 文件结尾，常用的关键字如下

  • syntax：执行 Protobuf 的版本，通常取值为 proto2 和 proto3
  • message：表示定义一个消息。
  • package：表示当前文件的包名

message 表示定义一个消息。在 C++ 中的表现形式为 类，如 message LoginRequest 就表示会定义一个 class LoginRequest

package 是包名，在 C++ 中的表现形式为 namespace，如 package login 就表示后续声明的消息都会保存在 namespace login 中

2. 生成 Protobuf 文件

对于 .proto 文件，需要使用 Protobuf 提供的编译器进行编译，即 protoc

  • 对于 C++，编译器会根据每个 生成一个 .h 和 .cc 文件 .proto，其中包含文件中描述的每种消息类型的类。
  • 对于Java，编译器会生成一个 .java 文件，其中包含每个消息类型的类，以及Builder 用于创建消息类实例的特殊类。
