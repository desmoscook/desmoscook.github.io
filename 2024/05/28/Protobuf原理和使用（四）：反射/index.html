什么是反射机制

所谓反射机制，就是计算机程序在运行时可以访问，检测和修改它自身行为或状态的机制。

反射机制是建立在 元信息 的基础上的

元信息：即系统自描述信息，用于描述系统本身。举例来讲，即系统有哪些类？类中有哪些字段、哪些方法 ？字段属于什么类型、方法又有怎样的参数和返回值？

反射机制的使用场景

反射机制由于可以在运行时动态的获取对象信息的类型和信息，所以可以用于 动态类型检查和类型转换

而实际上，对于 Protobuf，不仅可以用于序列化和反序列化，也可以用于配置文件的管理。

由于 Protoubf 会自动生成 .proto 对应的类，所以通过 Protobuf 的 IDL 文件来声明一个配置文件标准，不需要再单独进行对应配置类的实现，可以节省大量的时间。然后反射机制来实现统一的配置文件读取即可实现配置文件的管理。

Protobuf反射相关类和API

描述符 方法
FileDescriptor 获取 Proto 文件中的 Descriptor 和 ServiceDescriptor
Descriptor 获取类message属性和方法，包括FieldDescriptor和EnumDescriptor
FieldDescriptor 获取message中各个字段的类型、标签、名称等
EnumDescriptor 获取Enum中的各个字段名称、值等
ServiceDescriptor 获取service中的MethodDescriptor
MethodDescriptor 获取各个RPC中的request、response、名称等

1. google::protobuf::Message

Message 是所有 Protobuf 自定义类型的基类，继承自 MessageLite，其中 MessageLite 更加轻量级。

Message 提供了两个接口，用于获取当前 Message 的反射相关的类

2. google::protobuf::Descriptor

Descriptor 是针对Message 类型定义的描述，包括 Message 的名字，所有字段的描述，以及原始 proto 文件内容
