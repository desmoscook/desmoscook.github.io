    1. 定义 Protobuf 数据结构（IDL）

    Protobuf 的 IDL 文件以 .proto 文件结尾，常用的关键字如下

    • syntax：执行 Protobuf 的版本，通常取值为 proto2 和 proto3
    • message：表示定义一个消息。
    • package：表示当前文件的包名

    message 表示定义一个消息。在 C++ 中的表现形式为 类，如 message LoginRequest 就表示会定义一个 class LoginRequest

    package 是包名，在 C++ 中的表现形式为 namespace，如 package login 就表示后续声明的消息都会保存在 namespace login 中

    2. proto文件数据类型

    .proto 文件对于每个字段都会有一个数据类型，如 C++ 中的 int，double。protobuf支持的数据类型如下

    Protobuf type desc C++ type
    double 64位浮点数 double
    float 32位浮点数 float
    int32 32位有符号整数 int32_t
    int64 64位有符号整数 int64_t
    uint32 32位无符号整数 uint32_t
    uint64 64位无符号整数 uint64_t
    sint32 32位有符号整数，使用ZigZag编码 int32_t
    sint64 64位有符号整数，使用ZigZag编码 int64_t
    fixed32 32位固定长度无符号整数 uint32_t
    fixed64 64位固定长度无符号整数 uint64_t
    sfixed32 32位固定长度有符号整数 int32_t
    sfixed64 64位固定长度有符号整数 int64_t
    bool 布尔值 bool
    string 字符串（UTF-8编码或7-bit ASCII） std::string
    bytes 任意字节序列 std::string
    enum 枚举类型 C++ 枚举类型
    message 嵌套消息类型 C++ 类（由protoc生成）

    如对于上述例子，name 和 pwd 在编译后的代码中 都是 std::string 类型，sucess 则是 bool 类型

    3. protobuf 字符修饰符

    修饰符 说明 使用场景 适用版本
    required 字段必须存在且被设置（已弃用） 重要字段必须设置时使用（proto2） proto2
    optional 字段是可选的，未设置时使用默认值 非必要字段，或有默认值的字段 proto2, proto3
    repeated 字段可以包含零个或多个同类型元素 需要表示多个值的字段 proto2, proto3

    在 proto3 中，required 字段被弃用，如果不设置 optional，默认字段必须被设置值

    4. protobuf API

    前面提到，proto 中的每个 message 都是一个 class，其中每个字段都是一个成员。

    而对于每个成员，protobuf 也会生成对应的 API 用来对数据进行操作。

    这里先给出示例的 .proto 文件，后续展示的所有 API 都是该文件编译生成的。

    4.1 数据操作和获取 API ：

    数据的操作和获取是针对数据成员而言的，每个数据成员都有一份

    • **Getter 方法 field()**：获取字段的值。
    • Setter 方法 set_field()：设置字段的值。
    • **has 方法 has_field()**：检查字段是否设置（主要用于 optional 字段）。
    • **clear  方法 clear_field()**：清除字段的值，将其重置为默认值。

    4.2 序列化和反序列化 API：

    序列化 api 是针对类而言的，一个类只有一个，用于将类的所有变量序列化。

    序列化

    • **SerializeToString(std::string* output)**：将消息序列化为一个字符串，并将结果存储在 output 中。
    • **SerializeToArray(void* data, int size)**：将消息序列化到一个数组中，data 是数组指针，size 是数组大小。
    • **SerializeToOstream(std::ostream* output)**：将消息序列化到一个输出流（如文件流）中。
    • **SerializeAsString()**：返回序列化后的字符串。

    反序列化

    • **ParseFromString(const std::string& data)**：从一个字符串中解析消息，data 是包含序列化消息的字符串。
    • **ParseFromArray(const void* data, int size)**：从一个数组中解析消息，data 是数组指针，size 是数组大小。
    • **ParseFromIstream(std::istream* input)**：从一个输入流中解析消息（如文件流）。
    • **ParsePartialFromString(const std::string& data)**：从一个字符串中部分解析消息，可以处理部分字段缺失的情况。
    • **ParsePartialFromArray(const void* data, int size)**：从一个数组中部分解析消息，可以处理部分字段缺失的情况。

    序列化大小获取

    • **ByteSize()**：返回消息序列化后的字节大小
    • **ByteSizeLong()**：在较新的版本中，返回消息序列化后的字节大小，支持更大的消息

    4.3 调试 API

    • **DebugString()**：返回消息的调试字符串表示
    • **ShortDebugString()**：返回消息的简短调试字符串表示
    • **PrintDebugString()**：打印消息的调试字符串表示
    • **Utf8DebugString()**：返回消息的 UTF-8 编码的调试字符串表示。

    4. 生成 Protobuf 文件

    对于 .proto 文件，需要使用 Protobuf 提供的编译器进行编译，即 protoc

    • 对于 C++，编译器会根据每个 生成一个 .h 和 .cc 文件 .proto，其中包含文件中描述的每种消息类型的类。
    • 对于Java，编译器会生成一个 .java 文件，其中包含每个消息类型的类，以及Builder 用于创建消息类实例的特殊类。
